以下内容建议在视频中谨慎表述：

- `10Gbps`

- `SNMP`
  - 当前更准确的说法是：`支持 SNMP 流量识别与统计`
  - 不建议直接说成：`已完成完整 SNMP 设备监控平台`

- `在线真实抓包`
  - 当前版本更准确的表述是：`基于离线抓包文件进行实时化回放演示`
  - 这样更稳妥，也更符合你现有实现

## 3. 推荐的视频演示主线

建议不要按页面逐个介绍，而是按“发现问题 -> 定位问题 -> 研判问题”的业务链路来讲。

推荐主线如下：

1. 先展示总览页，说明系统能够实时看到核心网络指标
2. 再展示实时监控和 Top-K，说明系统能够发现重点通信对象
3. 然后展示威胁告警和地图溯源，说明系统能够识别异常并定位来源
4. 再展示历史检索和 IP 画像，说明系统支持追踪与复盘
5. 最后展示 AI 助手，说明系统具备智能化辅助研判能力

这条主线最适合比赛答辩，因为它强调的是完整闭环：

`抓包数据 -> 协议识别 -> 指标统计 -> Top-K -> 告警检测 -> 地图溯源 -> 历史检索 -> IP 画像 -> AI 分析`

## 4. 4 分钟演示视频脚本

### 片段 1：项目开场

时间建议：`00:00 - 00:20`

画面操作：

- 展示系统首页或登录后主界面
- 镜头停在整体大屏

讲解词：

大家好，我们的作品是智慧校园网络安全态势感知系统。系统面向校园网络流量监测场景，能够对抓包数据进行解析、识别和可视化展示，并实现 Top-K 流量排行、异常告警、地理溯源、历史检索以及 AI 智能研判。

### 片段 2：总览页核心指标

时间建议：`00:20 - 00:45`

画面操作：

- 停留在总览页
- 依次指向实时吞吐量、累计威胁数、独立攻击源、AI 引擎状态

讲解词：

在总览页中，我们首先展示赛题要求中的核心指标，包括实时吞吐量、攻击事件数量、独立攻击源数量，以及智能分析引擎状态。系统支持以 `Mbps`、`Bytes`、`Packets`、`PPS` 等多种指标反映网络运行状态。

### 片段 3：协议画像与区域透视

时间建议：`00:45 - 01:10`

画面操作：

- 展示总览页的流量趋势图
- 展示协议画像图
- 展示区域流量透视图

讲解词：

同时，系统对网络流量进行协议识别与聚合分析，可以直观看到 HTTP、HTTPS、SSH、DNS、IMAP、SNMP 等协议的流量占比，并按区域进行透视，帮助管理者快速了解当前网络活动分布。

### 片段 4：瞬时活跃监控

时间建议：`01:10 - 01:35`

画面操作：

- 切换到“瞬时活跃监控”
- 展示双图表
- 滚动查看活跃链路表

讲解词：

在瞬时活跃监控模块中，系统按秒级切片展示当前最活跃的通信链路，包含源 IP、目标 IP、源区域、吞吐量、包速率以及识别出的应用协议。这一部分对应了赛题中对 IP 级流量分析和实时监测的要求。

### 片段 5：Top-K 排行追踪

时间建议：`01:35 - 02:00`

画面操作：

- 切换到“大流排行追踪”
- 展示 Top-10 饼图、柱状图和明细表

讲解词：

在 Top-K 模块中，系统对当前和累计流量进行排名，支持从图形和表格两个维度观察全网流量大户。用户不仅可以看到前十节点的占比，还可以看到每个节点的协议特征、包数和累计流量，从而快速定位重点对象。

### 片段 6：威胁告警检测

时间建议：`02:00 - 02:30`

画面操作：

- 切换到“威胁告警雷达”
- 展示雷达图和告警表
- 点开一条重点告警

讲解词：

在安全告警模块中，系统基于启发式规则对异常流量进行检测，目前可识别 DDoS、端口扫描、蠕虫传播、异常爬虫和数据外泄等行为。每条告警都包含攻击源 IP、目标 IP、来源区域、流量大小、危险等级等信息，实现从流量监测到风险发现的闭环。

### 片段 7：地图溯源

时间建议：`02:30 - 02:55`

画面操作：

- 展示地图溯源模块
- 放大校园地图
- 点击某个告警点

讲解词：

为了增强展示效果和业务语义，我们将抽象的 IP 地址映射到校园区域和楼宇位置，在地图上直观标注威胁来源。这样不仅能看到“有攻击”，还能看到“攻击来自哪里”，更符合智慧校园场景下的安全态势感知需求。

### 片段 8：历史检索

时间建议：`02:55 - 03:20`

画面操作：

- 切换到“历史溯源检索”
- 输入某个 IP
- 演示筛选和检索结果

讲解词：

在历史检索模块中，系统能够对近期流量和告警进行缓存和检索，支持按 IP、协议、状态、区域进行筛选。这一能力有助于管理者对异常行为进行追踪和复盘，而不仅仅停留在当前时刻的观测。

### 片段 9：独立 IP 画像

时间建议：`03:20 - 03:45`

画面操作：

- 进入“独立 IP 画像”
- 展示风险等级、总流量、协议分布、端口分布、近期告警

讲解词：

对于重点对象，系统还提供独立 IP 画像功能。这里可以看到该 IP 的风险等级、累计告警数、关联对象、主要协议、重点端口以及近期行为趋势，实现从宏观监测到微观分析的下钻。

### 片段 10：AI 智能分析

时间建议：`03:45 - 04:10`

画面操作：

- 打开 AI 面板
- 输入“分析当前最危险的 IP”或“分析这个异常区域”

讲解词：

在智能分析方面，系统接入了大模型能力。AI 助手不仅能回答问题，还会结合当前 Top-K、异常区域、最新告警和 IP 历史进行辅助研判，帮助用户更快理解当前风险态势。这也是本项目相较于传统可视化看板的一个亮点。

### 片段 11：总结收尾

时间建议：`04:10 - 04:25`

画面操作：

- 回到总览页
- 全局镜头收尾

讲解词：

总体来看，本系统完成了从抓包数据解析、协议识别、指标统计，到 Top-K 排行、异常检测、地图溯源、历史追踪和智能研判的完整链路，较好对应了赛题对于网络流量监测与安全分析的核心要求。

### 片段 12：结束页

时间建议：`04:25 - 04:35`

画面操作：

- 展示项目标题页或系统 LOGO

讲解词：

以上就是我们的演示，谢谢各位老师观看。
