# 智慧校园态势感知系统项目框架与前后端技术亮点

## 1. 项目定位

本项目是一个面向智慧校园网络场景的可视化态势感知与安全分析平台，核心目标是对校园网络流量进行采集、解析、告警识别、区域映射、实时展示与智能问答分析，形成“抓包数据 -> 后端分析 -> 前端大屏 -> AI 研判”的完整闭环。

项目既能展示校园网络实时运行状态，也能对异常流量和威胁告警进行可视化追踪，并通过内置 AIOps 智能助手帮助使用者快速理解当前风险态势。

---

## 2. 项目整体架构

系统采用前后端分离架构，整体由四部分组成：

1. 数据采集与解析层
   - 基于离线 PCAP 抓包文件进行流量解析
   - 对网络流量进行协议识别、流聚合、异常识别

2. 后端分析与服务层
   - 基于 FastAPI 提供 HTTP API 与 WebSocket 实时推送
   - 负责流量状态维护、Top-K 统计、区域分析、告警缓存、AI 上下文构建

3. 前端可视化展示层
   - 基于 Vue 3 构建 Clay 风格智慧校园大屏
   - 提供总览、实时监控、排行追踪、历史检索、威胁雷达、地图溯源等模块

4. 智能研判层
   - 接入大模型对话能力
   - 支持基于系统实时状态的多轮上下文问答和工具调用分析

## 4. 系统核心工作流程

本系统的核心工作链路如下：

### 4.1 数据处理流程

1. 后端启动时加载 PCAP 文件
2. `engine.py` 对抓包数据进行逐包解析
3. 提取源 IP、目的 IP、字节数、包数、协议特征等信息
4. 按时间窗口进行流量聚合
5. 形成：
   - 当前流量 `current_flow`
   - Top-K 排行 `topk_flow`
   - 当前告警 `current_alerts`
6. 同时将最近告警与流量写入历史缓冲

### 4.2 数据分发流程

1. `routers.py` 通过 `/ws/flow` 每秒向前端推送实时流量和告警
2. 前端主应用 `App.vue` 接收消息后进行统一状态更新
3. 各个页面组件根据当前页面职责进行独立展示

### 4.3 智能分析流程

1. 用户在 `Chat.vue` 中输入问题
2. 前端通过 `/ws/chat` 将问题发送到后端
3. 后端基于当前系统快照构建结构化上下文
4. 大模型可结合工具调用查询：
   - Top-K
   - 区域统计
   - 系统资源
   - 最新告警
   - 指定 IP 历史
5. 模型输出简洁、专业的分析结果返回前端展示

## 5. 后端技术亮点

### 5.1 FastAPI + 生命周期任务管理

项目后端采用 FastAPI 作为服务框架，具备以下优势：

- 开发效率高，接口定义清晰
- 同时支持 HTTP 与 WebSocket
- 易于扩展异步任务与实时推送

在 `main.py` 中使用 `lifespan` 管理服务启动和退出过程，这样可以在应用启动时自动加载抓包解析线程与异步消费者任务，保证系统具备工程化的生命周期控制能力。

### 5.2 抓包解析与滑动窗口聚合

在 `engine.py` 中，系统基于抓包文件对网络数据包进行持续解析，并按时间窗口进行聚合，形成接近实时的动态视图。

技术特点：

- 支持多种协议类型识别
- 对源/目的 IP 建立流级别聚合
- 输出吞吐量、包数、协议特征等关键指标
- 通过队列缓冲降低前后端耦合度

这一设计使系统具备“离线数据源 + 实时展示效果”的能力，适合演示、答辩和教学场景。

### 5.3 全局状态中心设计

在 `core.py` 中，系统设计了集中式全局状态：

- `current_flow`
- `topk_flow`
- `current_alerts`
- `recent_alerts`
- `flow_history`

这使得系统具备两个优势：

1. 实时查询统一从一个状态中心读取，前后端语义一致
2. AI 对话与业务图表共享同一套实时数据源，避免口径不一致

### 5.4 校园内外网区域映射机制

系统并不仅展示 IP 本身，而是引入了校园区域语义。

在 `core.py` 中，通过：

- 校园内部 IP 前缀池
- 楼宇与区域映射表
- 稳定哈希落点机制

将网络流量映射到：

- 学生宿舍区
- 教学区
- 行政区
- 数据中心
- 运动区
- 校园出口

这样做的亮点是：把抽象的网络流量数据转化为具备校园空间含义的业务画像，显著增强了展示效果与答辩说服力。

### 5.5 AIOps 智能问答与工具调用机制

当前版本的 AI 模块不是简单聊天，而是一个具备上下文和检索能力的轻量级智能体。

后端亮点包括：

- 每个 WebSocket 连接维护独立会话历史
- 保留最近多轮消息上下文
- 动态构建系统快照作为结构化 prompt
- 支持工具调用：
  - `get_topk`
  - `get_zone_stats`
  - `get_system_stats`
  - `get_latest_alerts`
  - `get_ip_history`

这意味着 AI 回答不再是泛泛而谈，而是能够结合当前真实系统状态进行针对性研判。

## 6. 前端技术亮点

### 6.1 Vue 3 组件化大屏架构

前端采用 Vue 3 进行组件化开发，按业务职责拆分为多个视图和复用组件：

- 总览组件
- 实时监控组件
- 排行组件
- 历史组件
- 告警组件
- 地图组件
- AI 对话组件

组件化架构的优点是：

- 模块边界清晰
- 维护成本低
- 便于快速增加新页面或替换单一模块

### 6.2 大屏式多视图看板设计

项目不是传统后台表格系统，而是面向展示与答辩的多视图大屏。

用户可以通过左侧导航在多个分析视角中切换：

- 总体态势
- 实时活跃链路
- Top-K 排行
- 历史检索
- 威胁告警雷达

这种设计更适合面向评审专家进行模块化介绍，展示系统的层次与完整性。

### 6.3 Clay 风格与毛玻璃视觉系统

前端采用统一的 Clay 风格视觉语言，结合毛玻璃质感、圆角卡片、柔和阴影与状态色分层，提升系统辨识度和专业感。

视觉层面亮点包括：

- 全局设计变量统一管理
- 侧边栏、卡片、图表、地图、聊天面板风格统一
- Hover、展开、抽屉、弹窗等交互具备较强一致性

这使得系统不只是“能用”，而且“好看、统一、适合答辩展示”。

### 6.4 WebSocket 驱动的实时更新

在 `App.vue` 中，前端通过 WebSocket 接收后端每秒推送的数据，并将其分发到各页面组件。

技术亮点：

- 实时更新，不依赖频繁轮询
- 数据接收后进行帧级合并刷新，减少页面卡顿
- 支持高频动态图表和告警流更新

这使系统具备实时态势感知平台应有的动态感。

### 6.5 ECharts 多图表联动展示

项目中广泛使用图表来表达网络态势，包括：

- 折线图：吞吐量趋势
- 条形图：区域统计、流量强度
- 饼图：协议画像、Top-K 占比
- 雷达图：威胁画像

亮点在于：

- 图表和业务页面强绑定
- 支持悬停提示与交互反馈
- 通过异步加载优化首屏性能

### 6.6 校园地图安全溯源可视化

`MapTraceability.vue` 是本项目非常有展示力的模块。

其技术亮点包括：

- 攻击点按校园建筑或边界区域稳定映射
- 内外部流量颜色区分
- 支持局部预览与全屏查看
- 支持点位悬停详情联动

这类地图式展示能将安全告警从“日志表”升级为“空间化可视分析”，非常适合答辩场景。

### 6.7 AI 聊天抽屉与会话体验优化

前端的 AI 模块并不是普通输入框，而是一个集成在主应用中的智能分析抽屉。

当前亮点包括：

- 抽屉式 AI 面板，节省主界面空间
- 支持新建对话与清空上下文
- 支持查询中状态提示

这体现了项目在“智能化交互”方向上的设计深度。
